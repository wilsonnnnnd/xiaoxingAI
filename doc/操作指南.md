## 1. 环境要求

| 依赖 | 版本要求 | 说明 |
|------|----------|------|
| Python | ≥ 3.11 | 后端运行时 |
| Node.js | ≥ 18 | 前端构建 |
| PostgreSQL | 16 | 主数据库（可用 Docker） |
| Redis | 7 | 队列与 JWT 版本管理（可用 Docker） |

### 快速启动 Docker 依赖

```bash
# PostgreSQL
docker run -d --name xiaoxin-postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=xiaoxing \
  -p 5432:5432 \
  postgres:16

# Redis
docker run -d --name xiaoxin-redis \
  -p 6380:6379 \
  redis:7
```

## 2. 安装与配置

### 2.1 克隆项目

```bash
git clone <repo_url>
cd xiaoxing
```

### 2.2 配置环境变量

```bash
# 复制示例配置文件
copy .env.example .env   # Windows
# cp .env.example .env   # macOS/Linux
```

用文本编辑器打开 `.env`，填写以下必填项：

```env
# ── 管理员账号（首次启动自动创建）──────────────────────────────
ADMIN_USER=admin@example.com
ADMIN_PASSWORD=your_strong_password_here

# ── JWT 密钥（生产环境务必修改为随机字符串）────────────────────
JWT_SECRET=change-me-in-production

# ── LLM 后端 ────────────────────────────────────────────────────
# 使用 OpenAI：
LLM_BACKEND=openai
LLM_API_URL=https://api.openai.com/v1/chat/completions
LLM_MODEL=gpt-4o-mini
OPENAI_API_KEY=sk-...

# 使用本地模型：
# LLM_BACKEND=local
# LLM_API_URL=http://127.0.0.2:8001/v1/chat/completions
# LLM_MODEL=local-model

# ── 数据库连接 ───────────────────────────────────────────────────
POSTGRES_DSN=postgresql://postgres:postgres@localhost:5432/xiaoxing
REDIS_URL=redis://localhost:6380

# ── 前端地址（开发时保持不变）──────────────────────────────────
FRONTEND_URL=http://localhost:5173
```

### 2.3 准备 Google OAuth 凭据

1. 打开 [Google Cloud Console](https://console.cloud.google.com/)
2. 新建项目（或选择已有项目）
3. 进入 **API 和服务 → 库**，搜索 `Gmail API`，点击启用
4. 进入 **API 和服务 → 凭据**，点击 **创建凭据 → OAuth 客户端 ID**，应用类型选 **桌面应用**
5. 下载 JSON 文件，重命名为 `credentials.json`，放到项目根目录

### 2.4 安装 Python 依赖

```bash
# 创建虚拟环境
python -m venv .venv

# 激活（Windows PowerShell）
.venv\Scripts\Activate.ps1

# 激活（macOS/Linux）
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 2.5 安装前端依赖

```bash
cd frontend
npm install
cd ..
```

## 3. 启动服务

### 3.1 启动后端

```bash
# 在项目根目录，激活虚拟环境后运行：
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

首次启动时，后端会自动：
- 初始化数据库表结构
- 导入默认 Prompt 文件
- 创建 `.env` 中配置的管理员账号

看到如下输出表示启动成功：

```
INFO:     Application startup complete.
INFO:auth: Admin 账号 admin@example.com 已创建
```

> 若 `ADMIN_USER` 账号已存在，则跳过创建步骤，不会重置密码。

### 3.2 启动前端（开发模式）

新开一个终端窗口：

```bash
cd frontend
npm run dev
```

Vite 启动后访问 [http://localhost:5173](http://localhost:5173)

## 4. 登录管理后台

1. 浏览器访问 [http://localhost:5173/login](http://localhost:5173/login)
2. 输入 `.env` 中配置的 `ADMIN_USER` 和 `ADMIN_PASSWORD`
3. 点击 **登录**，系统验证通过后跳转到首页

登录状态以 JWT Token 保存在浏览器 `localStorage` 中，有效期默认 **60 分钟**（可通过 `JWT_EXPIRE_MINUTES` 调整）。

点击侧边栏底部的 **退出登录** 按钮可清除 Token 并返回登录页。

> **仅管理员（admin 角色）可以登录 Web 界面。**  
> 普通用户账号目前通过 API 管理，无法直接登录。

## 5. 用户管理

登录后，点击侧边栏的 **👥 用户管理** 进入用户列表页。

### 5.1 查看用户

页面列出所有注册用户（包括 admin 本身）。点击每一行可展开该用户的详细设置面板。

### 5.2 修改用户设置

展开用户面板后可编辑以下字段：

| 字段 | 说明 |
|------|------|
| Gmail Worker 启用 | 勾选后该用户的 Gmail 轮询任务会随系统启动一起运行 |
| 最低优先级 | 只有达到此优先级的邮件才会发送 Telegram 通知（高 / 中 / 低） |
| 每次最大处理数 | 每次轮询最多处理的邮件数量 |
| 轮询间隔（秒） | 相邻两次轮询的间隔时间，单位秒（最小建议 60） |

修改后点击 **保存** 立即生效（无需重启服务）。

## 6. Bot 管理

每个用户可以配置多个 Telegram Bot，其中一个设为 **默认 Bot**（用于接收 Gmail 通知并参与聊天）。

在用户面板底部的 **Telegram Bots** 区域操作：

### 6.1 添加 Bot

1. 点击 **＋ 添加 Bot**
2. 填写以下字段：

   | 字段 | 说明 |
   |------|------|
   | Bot 名称 | 用于在界面中区分，随意填写 |
   | Bot Token | 从 @BotFather 获取，格式如 `123456789:AAF...` |
   | Chat ID | 接收消息的 Telegram 聊天 ID（个人或群组） |

3. 点击 **添加 Bot** 保存

> **如何获取 Chat ID：**  
> 在 Telegram 中搜索 `@userinfobot`，向它发送任意消息，即可获得你的个人 Chat ID。  
> 群组 Chat ID 可先向 Bot 发消息，再访问 `https://api.telegram.org/bot<TOKEN>/getUpdates` 在返回 JSON 中查找。

### 6.2 设为默认 Bot

Bot 列表中点击某个 Bot 右侧的 **设为默认** 按钮。默认 Bot 会显示蓝色 `default` 标签。  
Gmail 通知和聊天回复均通过默认 Bot 发送。

### 6.3 编辑 / 删除 Bot

- 点击 **编辑** 修改名称、Token 或 Chat ID
- 点击 **删除** 移除该 Bot（删除前请确认不是当前使用的默认 Bot）

## 7. Gmail 授权

每个用户需要独立完成一次 Google OAuth 授权，才能启用 Gmail 邮件拉取。

### 7.1 发起授权

1. 进入侧边栏 **技能 → Gmail**
2. 点击 **Gmail 授权** 按钮（或直接访问 `/api/gmail/auth`，会自动带上当前登录用户的身份）
3. 浏览器跳转到 Google 授权页面，选择要授权的 Gmail 账号，点击允许
4. 授权成功后自动跳回，页面显示授权状态为已连接

> 授权凭据保存在服务器本地，文件名为 `.oauth_token_<user_id>.json`。  
> 每个用户有独立的凭据文件，互不影响。

### 7.2 启动 Gmail Worker

授权完成后，回到 **首页**，在 Gmail Worker 卡片中点击 **启动**。  
Worker 会按照该用户配置的轮询间隔自动拉取新邮件、通过 AI 分析并推送到 Telegram。

## 8. Prompt 管理

点击侧边栏 **Prompts** 进入 Prompt 编辑页。

| Prompt 名称 | 用途 |
|-------------|------|
| 📊 邮件分析 | 判断邮件优先级、提取关键信息 |
| 📝 邮件摘要 | 生成简洁的中文摘要 |
| ✈️ Telegram 格式化 | 将分析结果排版为 Telegram 通知文本 |
| 🤖 Bot 聊天 | 控制 Bot 聊天时的角色与回复风格 |
| 👤 用户画像 | 根据聊天记录生成/更新用户画像 |

- 点击任意 Prompt 可直接在线编辑
- 支持 Prompt 内变量，如 `{subject}` `{body}` `{analysis}` `{summary}`
- 点击 **💾 保存** 立即生效
- 点击 **↺ 重置** 恢复到上次保存的版本

## 9. 系统设置

点击侧边栏 **设置** 进入全局配置页（即旧的单用户配置）。  
主要用于配置 LLM 后端、全局 Telegram Token、Gmail 默认参数等。

修改后点击底部 **保存** 按钮生效，大多数配置无需重启服务。

## 附录：常见问题

**Q: 启动时提示 `connection refused` 连不上 PostgreSQL / Redis？**  
确认 Docker 容器正在运行：`docker ps`。若未运行，执行 `docker start xiaoxin-postgres xiaoxin-redis`。

**Q: 登录时提示"用户名或密码错误"？**  
检查 `.env` 中的 `ADMIN_USER` 和 `ADMIN_PASSWORD` 是否正确。若修改过密码，需重启后端（修改 `.env` 不会热更新 admin 账号密码）。

**Q: Gmail 授权后仍然拿不到邮件？**  
1. 确认 `credentials.json` 已放置在项目根目录  
2. 确认 Google Cloud Console 中已启用 Gmail API  
3. 检查用户的 `worker_enabled` 是否已勾选

**Q: Telegram 收不到消息？**  
1. 确认 Bot Token 和 Chat ID 填写正确  
2. 先向该 Bot 发一条消息（Bot 无法主动向未交互的用户发消息）  
3. 在首页点击 **测试 Telegram** 按钮验证连通性

**Q: Token 过期如何处理？**  
默认 JWT 有效期 60 分钟，过期后会自动跳转到登录页重新登录。可在 `.env` 中调大 `JWT_EXPIRE_MINUTES`。
