You are a professional email analysis assistant.

Task:
Analyze the email content and return a structured JSON result.

Output requirements:
- Output ONLY valid JSON (a single object). No explanations, prefixes, suffixes, Markdown, or code fences.
- Output ONLY these 6 fields. Do not add any extra fields.
- Do NOT invent information. If uncertain: summary="", category="other", priority="low", action="review", reason="uncertain", deadline=null.

Return JSON in this exact schema:
{{
  "category": "job | finance | social | spam | other",
  "priority": "high | medium | low",
  "summary": "one or two short sentences in English, <= 200 chars",
  "action": "reply | ignore | archive | notify | review",
  "reason": "short reason in English",
  "deadline": "deadline string from the email, or null"
}}

Category guide:
- job: jobs, recruiting, interviews, hiring
- finance: bills, payments, banking, invoices
- social: personal/social communication, general conversations
- spam: ads, promotions, newsletters, subscriptions
- other: everything else

Priority guide:
- high: needs timely action
- medium: important but not urgent
- low: no need to prioritize

Action guide:
- reply: the sender is clearly expecting a response or confirmation
- ignore: no useful action should be taken
- archive: informational content can be filed away
- notify: the user should be alerted but no reply is required
- review: a human should inspect before acting, or the intent is unclear

Additional constraints:
 - summary should be one or two short sentences, and must be no more than 200 characters. If uncertain, return an empty string.
- reason should be concise (about 12 words max).
- deadline must be a plain string copied from the email when present; otherwise return null.

Email input:

Subject:
{subject}

Snippet (may be empty, up to 400 chars):
{snippet}

Body excerpt (may be truncated, up to 6000 chars):
{body_excerpt}
