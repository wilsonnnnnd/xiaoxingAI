# 用户记忆系统

## 概述

每个 Bot 为与之聊天的用户构建并维护一套结构化长期记忆。记忆从对话历史中自动提炼，分类存储，并在后续对话中按相关性注入，实现个性化回复。

## 记忆分类

| 分类 | 存储内容 |
|------|---------|
| `[事实]` | 客观信息：姓名、职业、所在地、年龄、家庭情况 |
| `[偏好]` | 喜好、习惯、偏好的回复风格、不喜欢的事物 |
| `[近期事件]` | 用户近期提到的事情，尽量带时间感（如"最近"、"上周"） |
| `[性格观察]` | 从对话中观察到的性格特点，须有对话依据，不随意推测 |

## 记忆构建流程

1. **积累** — 每轮对话追加到 `_histories_today`（内存 + Redis 双存）
2. **提炼** — 每天凌晨，使用 `user_profile.txt` Prompt 将当天历史送入 `build_user_profile()`
3. **合并** — LLM 将新观察与已有记忆合并，删除明显过时的条目
4. **持久化** — 更新后的记忆写入 PostgreSQL `user_profile` 表

Prompt 强制执行的规则：
- 只记录用户明确说过或强烈暗示的内容，禁止主观推测
- 每条记忆不超过 30 字
- 合并时删除过时条目

## 注入时的相关性筛选

系统不会每轮都全量注入记忆，而是按相关性过滤：

- **事实、偏好、性格观察** — 始终注入
- **近期事件** — 仅当当前消息与事件文本有 2 个以上公共字符时注入

这样既保持上下文窗口精简，又能在相关对话中提供有用背景。

## 手动触发

调试页面提供 "立即更新画像" 按钮，使用当前对话窗口历史作为输入，即时生成新记忆（无需等到凌晨）。

## 存储结构

- 表：`user_profile (bot_id, profile TEXT, updated_at)`
- 每个 Bot 对应一行
- `_histories_today` 由 Redis 持久化，凌晨更新后自动清空
